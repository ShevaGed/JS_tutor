Оператори JS:
оператор + додає об'єкти між собою рядки і числа якщо поєднується рядок і число він перетворює його на рядок: console.log("arr" + " - object"); === "arr - object"
якщо оператор + стоїть перед рядком він перетворює його на число, такий + називається унарним: console.log(4 + +"5"); === 9
оператор ++ інкримент додає 1 до об'єкту. може ставитись як перед(префіксне) значенням так і після(постфіксне): let incr = 10; incr++; === 11 або ++incr; === 11
оператор -- декримент віднімає 1 від об' єктуможе ставитись як перед(префіксне) значенням так і після(постфіксне) let decr = 10; decr--; === 9 або --decr; === 9 .якщо інкримент та декримент використовувати в постфіксній формі при визові значення відразу, то повернеться першочергове значення а не нове, а якщо в префіксній формі то повернеться відразу нове значення: let incr = 10; console.log(incr++); === 10; console.log(++incr); === 11
оператор % повертає залишок від ділення: console.log(5%2); === 1
оператор == порівнює два значення і повертає булінове значення console.log(2*4 == '8'); === true але такий спосіб не порівнює типи данних, тобтр якщо 8 це рядок воно поверне теж true
оператор === порівнює два значення і повертає булінове значення console.log(2*4 == '8'); === false і він порівнює тип данних
оператор && це та:  true && false === false (ЦЕЙ ОПЕРАТОР ПОВЕРТАЄ ПЕРШЕ НЕПРАВДИВЕ ЗНАЧЕННЯ А НЕ false ТА ОСТАННЄ ПРАВДИВЕ ЗНАЧЕННЯ ЯКЩО ВСІ ЗНАЧЕННЯ true)
оператор || це або: true || false === true  (ЦЕЙ ОПЕРАТОР ПОВЕРТАЄ ПЕРШЕ ПРАВДИВЕ ЗНАЧЕННЯ А НЕ true ТА ОСТАННЄ НЕПРАВДИВЕ ЗНАЧЕННЯ ЯКЩО ВСІ ЗНАЧЕННЯ false)
оператор ! це не: true && !false === true. особливість коли цей опертор поєднується з оператором порівняння то стає менше не один знак рівно: 2 + 2 * 2 != 6; === false (ЦЕЙ ОПЕРАТОР ПОВЕРТАЄ БУЛІНОВЕ ЗНАЧЕННЯ true або false)
(\n) - це символ переноса рядка
оператор += - це додавання з присвоєнням х = 3 х += 4 х = 7 (для рядків робить коннектикацію)



Умови в JS:

це команди if та else в яких такий синтаксис:

if (4 == 9) {
    console.log('Ok');
} else {
    console.log('Error');
}

інший варіант запису перевірки умов такий(і називається тернарним оператором, бо він перевіряє 3 умови. це єдиний такий оператор в js:

(num === 50) ? console.log('Ok') : console.log('Error');

умови можуть бути вкладеними (else if) і перевіряються таким чином: 

const num = 50;

if (num < 49) {
    console.log('Error');
} else if (num > 100) {
    console.log('Too much');
} else {
    console.log('Ok');
}

альтернативний варіант вкладених умов має такий синтаксис(його особливість він перевіряє точні порівняння, в ному не працюють оператори більше менше):

switch (num) {
case '49':
    console.log('Error');
    break;
case '100':
    console.log('Error');
    break;
case '50':
    console.log('Congretulation');
    break;
default:
    console.log('No way');
    break;
}


Цикли в JS:

перший цикл while найпростіше перевіряє чи виконується умова

let num = 50;
while (num <= 55) {
    console.log(num);
    num++;
}

другий цикл do while перевіряє чи виконується умова а потім запускає цикл

let num = 50;
do {
    console.log(num);
    num++;
}

while (num <= 55);

третій цикл for перевіряє чи виконується внутрішня умова яка складається з трьох аргументів(першого значення, умови коли закінчити цикл та дії з першим аргументом) а потім запускає цикл (ЦЕ ОДИН З НАЙВАЖЛИВІШИЙ З ЕЛЕМЕНТІВ В JS)

let num = 50;
for (let i = 1; i < 8; i++) {
    console.log(num);
    num++;
}

для цикклів є елементи преривання break який закінчує цикл як тільки отримає пеіне значення та  елемен continue який пропускає отримане значення та продовжує цикл з наступного

for (let i = 1; i < 10; i++) {
    if (i === 6) {
	//break;
	continue;
    }
    console.log(i);
}

Вкладені цикли виглядають так( є певна обмеженість на кількість вкладень циклів)

for (let i = 0; i < 3; i++) {
    console.log(i);
    for (let j = 0; j < 3; j++) {
	console.log(j);
    }
}

Приклад 2 з ялинкою (трикутник з зірочкою (астеріксом)

let result = '';
const length = 7;

for (let i = 1; i < length; i++) {
    
    for (let j = 0; j < i; j++) {
	result += "*";
    }
    result += '\n';
}
console.log(result);

Мітки у циклах виглядають так(вони потрібні для того щоб могли спрацьовувати деякі ключові слова в певному місці) в даному прикладі мітка це first:

first: for (let i = 0; i < 3; i++) {
    console.log(`First level: ${i}`);
    for (let j = 0; j < 3; j++) {
	console.log(`Second level: ${j}`);
	    for (let k = 0; k < 3; k++) {
		if (k === 2) continue first;
		console.log(`Third level: ${k}`);
	    }
    }
}


Принцип інтерполяцї дозволяє робити динамічний код (це корисно в буденних задачах js) це по суті коннектикація рдків певним чином.
Основний принцип це якщо потрібно використати десь змінну у вигляді рядка закконетувати її можна використавши тільки такі `` лапки і додавши знак $ перед рядком який коннектикується, а саму змінну взяти в {} дужки. Приклади:

const category = "toys";
console.log(`https://someurl.com/${category}/5`); 

const user = "Anton";
alert(`Hello, ${user}`);
